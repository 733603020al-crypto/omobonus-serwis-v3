# Инструкции по деплою на Vercel

## Проблема 404

Если сайт показывает ошибку 404 на Vercel, проверьте следующие настройки:

### 1. Структура проекта
Главная страница находится в: `src/app/page.tsx`
Это правильная структура для Next.js App Router.

### 2. Настройки проекта на Vercel

1. Перейдите в настройки проекта на Vercel
2. Проверьте раздел **"Root Directory"**:
   - Должно быть **пусто** или указан путь к корню репозитория
   - Если проект в подпапке, укажите путь к корню проекта

### 3. Настройки сборки

- **Framework Preset**: Next.js
- **Build Command**: `npm run build` (автоматически определяется)
- **Output Directory**: `.next` (автоматически определяется)
- **Install Command**: `npm install` (автоматически определяется)

### 4. Переменные окружения

Убедитесь, что в настройках проекта на Vercel добавлены все необходимые переменные окружения:
- `RESEND_API_KEY`
- `RESEND_FROM_EMAIL`
- `RESEND_TO_EMAIL`

### 5. Проверка сборки

1. Перейдите в раздел **"Deployments"**
2. Откройте последний деплой
3. Проверьте логи сборки на наличие ошибок
4. Если есть ошибки, исправьте их и перезапустите деплой

### 6. Пересборка

Если после исправлений сайт всё ещё показывает 404:
1. Перейдите в настройки проекта
2. Нажмите **"Redeploy"** на последнем деплое
3. Или сделайте новый коммит и пуш в репозиторий

## Важно

- Файл `vercel.json` **НЕ нужен** для Next.js проекта - Vercel автоматически определяет Next.js
- Убедитесь, что корневая директория проекта правильно указана в настройках Vercel
- Проверьте, что главная страница экспортируется как `export default function Home()`

## Контакты

Если проблема сохраняется:
1. Проверьте логи сборки на Vercel
2. Убедитесь, что проект собирается локально (`npm run build`)
3. Проверьте настройки корневой директории на Vercel
